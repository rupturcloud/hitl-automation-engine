# 🔧 Fix: Erro 500 em Produção — 2026-05-08

**Status**: ✅ RESOLVIDO  
**Data**: 2026-05-08 17:27 UTC  
**Causa**: Traefik (reverse proxy) não estava rodando

---

## 🔍 **DIAGNÓSTICO**

### Sintoma
```
GET https://app.ruptur.cloud/client-area/
→ HTTP 500 Internal Server Error (nginx)
```

### Investigação
1. ✅ Backend (gateway) estava ok: `curl localhost:3001/api/health` → 200 OK
2. ✅ Containers rodando: saas-web + warmup-runtime UP
3. ❌ **Traefik não estava rodando** (reverse proxy que faz proxy de 80/443)
4. ❌ Nginx legacy estava usando porta 80 (conflitando com Traefik)

### Raiz do Problema
- Traefik (configurado em `docker-compose-traefik.yml`) não foi iniciado
- Nginx legacy (máquina host) estava usando porta 80
- Quando clientes acessavam `app.ruptur.cloud`, Traefik não estava lá para fazer proxy
- Nginx retornava 500

---

## ✅ **SOLUÇÃO APLICADA**

### Passo 1: Parar Nginx Legacy
```bash
sudo systemctl stop nginx
sudo systemctl disable nginx
```

### Passo 2: Iniciar Traefik
```bash
cd /opt/ruptur
docker-compose -f docker-compose-traefik.yml up -d
```

### Resultado
```
✅ Traefik started (porta 80, 443)
✅ app.ruptur.cloud → HTTP 200
✅ saas.ruptur.cloud → HTTP 200
✅ /api/health → JSON válido
```

## 📝 **NOTAS TÉCNICAS**

### Por que Traefik?
- Traefik é um reverse proxy moderno que se integra com Docker via labels
- Lê labels dos containers (`traefik.enable`, `traefik.http.routers`, etc)
- Gerencia TLS/HTTPS automaticamente via Let's Encrypt
- Mais eficiente que Nginx manual para ambientes containerizados

### Configuração Traefik
```yaml
# Em docker-compose.yml (saas-web)
labels:
  traefik.enable: "true"
  traefik.docker.network: "ruptur-edge"
  traefik.http.routers.saas-web.rule: "Host(`app.ruptur.cloud`) || Host(`saas.ruptur.cloud`)"
  traefik.http.services.saas-web.loadbalancer.server.port: "4173"
```
